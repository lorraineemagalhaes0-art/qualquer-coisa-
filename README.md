Compartilhamento no desenvolvimento back-end refere-se a técnicas e arquiteturas utilizadas para permitir que múltiplos processos, serviços, servidores ou aplicações acessem os mesmos recursos, dados ou infraestrutura de forma segura e eficiente.

O GitHub é uma plataforma na nuvem baseada em Git, usada para hospedar, gerenciar e compartilhar projetos de código, sendo popularmente conhecida como uma "rede social para programadores". Ele facilita a colaboração entre desenvolvedores, permitindo o controle de versões, o trabalho em equipe e o uso de recursos como pull requests para revisar códigos. 

O GitLab, no contexto de programação backend, é uma plataforma abrangente de DevOps baseada em nuvem ou auto-hospedada, que centraliza o ciclo de vida do desenvolvimento de software, desde o controle de versão com Git até a automação de testes e implantação 

O Bitbucket é uma plataforma de hospedagem e gerenciamento de repositórios Git criada pela Atlassian, essencial para equipes de desenvolvimento backend colaborarem em código, realizar controle de versão, revisões (pull requests) e automação CI/CD. É muito utilizado para repositórios privados e integra-se nativamente com Jira e Trello. 

Hospedagem de backend é o serviço que fornece um servidor para armazenar e executar o código do lado do servidor (APIs, regras de negócio e bancos de dados), mantendo a aplicação 24/7 online. Diferente do frontend, ela processa lógica complexa, gerencia dados seguros e responde a requisições HTTP de clientes. 

Ferramentas de backend na programação são tecnologias (linguagens, frameworks, bancos de dados, servidores) usadas para construir a lógica, o gerenciamento de dados e a infraestrutura do lado do servidor. Elas processam requisições, interagem com o banco de dados e garantem a funcionalidade da aplicação. 

A gestão em programação backend refere-se à supervisão técnica e operacional do desenvolvimento do "lado do servidor" de uma aplicação web ou sistema. Diferente da gestão geral, foca na infraestrutura, bancos de dados, lógica de negócio e performance que sustentam a interface do usuário (frontend). 

Colaboração em programação backend refere-se ao conjunto de práticas, ferramentas e processos que permitem a múltiplos desenvolvedores trabalharem em harmonia na lógica de servidor, banco de dados e APIs de uma aplicação. 

Um Pull Request (PR), ou "solicitação de puxada" em tradução livre, é um mecanismo fundamental na programação, especialmente no desenvolvimento backend, usado para notificar os membros da equipe que uma funcionalidade, correção ou alteração de código foi concluída e está pronta para ser integrada ao repositório principal. 

Issue tracking (rastreamento de problemas) em programação backend é o processo sistemático de registrar, monitorar e gerenciar "issues" — pendências, bugs, erros de servidor, ou solicitações de novas funcionalidades — durante o ciclo de desenvolvimento de software. No contexto backend, isso é crucial para garantir a estabilidade de APIs, bancos de dados e regras de negócio, centralizando a comunicação sobre o que precisa ser consertado ou desenvolvido.
